# 札流し v2 仕様書

> ブランチ: `v2`  
> 最終更新: 2026-06-17  
> ステータス: レビュー済み（実装着手可）

## 1. 概要

### 1.1 目的

札流し練習 Web アプリ（[fudanagashi](https://kg9n3n8y.github.io/fudanagashi/)）の v2 改修として、以下を実現する。

- 練習の入り口となる**トップ画面**の追加
- **操作方法**（タップ / スワイプ）と**自動切り替えモード**の選択
- **和風テイストを保ったモダン UI** への刷新（レスポンシブ対応）
- **PWA 体験**の改善（初回インストール案内・自動アップデート）

### 1.2 対象外（v2 では行わない）

- ユーザーアカウント・クラウド同期
- 札の種類・枚数のカスタマイズ（100 枚固定を継続）
- オンライン対戦・ランキング

---

## 2. 現状（v1）の整理

| 項目 | 現状 |
|------|------|
| 技術スタック | HTML / CSS / バニラ JavaScript |

## 4. 機能要件

### 4.1 トップ画面

#### 4.1.1 概要

アプリ起動時のデフォルト画面とする。練習の開始と、直近の記録確認、ヘルプ・設定へのアクセスを提供する。

#### 4.1.2 表示要素

| 要素 | 説明 |
|------|------|
| アプリタイトル / ロゴ | 「札流し」ブランディング |
| 「練習を始める」ボタン | 主 CTA。練習画面へ遷移 |
| 直近の記録一覧 | 最大 5 件。新しい順 |
| ヘルプ入口 | アイコンまたはテキストリンク |
| 設定入口 | アイコンまたはテキストリンク |

#### 4.1.3 記録の保存

**保存タイミング:** 100 枚の練習が完了した時点（結果画面表示前または同時）

**保存先:** `localStorage`（ブラウザのローカルストレージ）

**保存件数:** 最大 5 件（6 件目以降は最古を削除）

**1 件あたりのデータ:**

```typescript
interface PracticeRecord {
  id: string;           // UUID またはタイムスタンプベースの一意 ID
  completedAt: string;  // ISO 8601（例: "2026-06-17T12:34:56.789Z"）
  elapsedMs: number;    // 経過時間（ミリ秒）
  mode: 'manual' | 'auto';  // 手動（タップ/スワイプ） or 自動
  inputMethod?: 'tap' | 'swipe';  // manual の場合のみ
  autoTotalSeconds?: number;      // auto の場合のみ（10〜180）
}
```

**一覧表示内容（例）:**

- 日時（相対表記可: 「今日 14:30」「昨日」など）
- 経過時間（`X分Y秒`）
- モード（手動 / 自動 10秒 など）

記録が 0 件のときは「まだ記録がありません」等のプレースホルダーを表示する。

#### 4.1.4 ヘルプ

トップ画面から開ける使い方ガイド。

**掲載内容（案）:**

1. 札流しとは何か（簡潔に）
2. 練習の流れ（鳥居 → 100 枚 → 結果）
3. 操作方法（タップ / スワイプの説明）
4. 自動モードの説明
5. 決まり字の見方
6. PWA としてホーム画面に追加するメリット（インストール案内への導線）

表示形式: モーダルまたはスライドインのオーバーレイ（画面遷移なしでも可）

### 4.2 操作方法の選択（タップ / スワイプ）

#### 4.2.1 設定項目

| 項目 | 値 | デフォルト |
|------|-----|-----------|
| 入力方式 | `tap` / `swipe` | `tap` |

設定は `localStorage` に永続化し、次回起動時も反映する。

#### 4.2.2 タップモード

v1 と同等の挙動。

| 状態 | タップ時の動作 |
|------|----------------|
| 鳥居表示中（未開始） | タイマー開始。1 枚目を表示 |
| 1〜99 枚目表示中 | 次の札へ即時切り替え。決まり字は非表示に戻す |
| 100 枚目表示中 | タイマー停止。結果画面へ |

#### 4.2.3 スワイプモード

指の動きに**自由追従**するカード UI。上下左右・斜めなど、ドラッグした方向に札がそのまま動く。

**レイアウト:**

- 前面: 現在の札（ドラッグ中は指の移動量に 1:1 で追従）
- 背面: 次の札（常に下に表示。ドラッグ開始時点で次札が見える）

**操作:**

| 操作 | 挙動 |
|------|------|
| ドラッグ開始 | 前面札が指に追従して移動。背面に次札を表示 |
| ドラッグ中（閾値未満） | 指を離すと元の位置にスナップバック |
| ドラッグ中（閾値以上） | 指を離すと、**ドラッグした方向**へスライドアウト。次札が前面に |
| 鳥居の開始 | タップまたはドラッグで開始（タップ併用） |

**方向:**

- 上下・左右・斜めなど**全方向**に対応
- 方向の制限は設けない。ユーザーが動かしたベクトルに沿って札が移動する
- スライドアウトも同じ方向へ飛ばす（例: 右上にドラッグ → 右上へ消える）

**閾値:**

ドラッグ開始位置からの**移動距離**（ユークリッド距離）で判定する。

```
distance = √(dx² + dy²)
```

| 条件 | 値（案） |
|------|----------|
| 確定閾値 | 画面短辺の 25% 以上 |

閾値未満の場合はスナップバック。実装時に UX テストで微調整する。

**アニメーション:**

- 追従: `transform: translate(x, y)` で 60fps 目標。`touch-action: none` でブラウザのスクロールと競合しない
- スナップバック: 200〜300ms の ease-out
- スライドアウト: ドラッグ方向へ 250〜400ms の ease-in。画面外まで移動後に次札へ切り替え

**制約:**

- ドラッグ中は背面札のタップ・ドラッグを受け付けない
- 決まり字表示中もドラッグで次へ進める（v1 同様、進行時に決まり字は非表示）
- 練習画面の縦スクロールは発生させない（`overflow: hidden`）

---

### 4.3 自動切り替えモード

#### 4.3.1 概要

一定間隔で自動的に次の札へ進むモード。手動（タップ/スワイプ）とは排他的に選択する。

#### 4.3.2 設定項目

| 項目 | 値 | デフォルト |
|------|-----|-----------|
| 練習モード | `manual` / `auto` | `manual` |
| 自動モード合計時間 | 10, 20, 30, …, 180（10 秒刻み） | 60 |

#### 4.3.3 切り替え間隔の計算

```
1枚あたりの間隔（秒） = 設定合計時間（秒） / 100
1枚あたりの間隔（ms）  = 設定合計時間（秒） × 10
```

| 設定合計時間 | 1 枚あたり |
|-------------|-----------|
| 10 秒 | 100 ms |
| 60 秒 | 600 ms |
| 180 秒 | 1,800 ms |

100 枚すべてが表示され終わると、設定した合計時間が経過していること。

#### 4.3.4 挙動

| フェーズ | 動作 |
|----------|------|
| 開始前 | 「開始」操作（タップまたはボタン）でタイマーと自動切り替えを開始 |
| 進行中 | 設定間隔ごとに次の札へ。手動操作は無効 |
| 100 枚完了 | 自動切り替え停止。結果画面へ |
| 中断 | 自動タイマーをクリア。確認後リセット |

**表示:**

- 進行中は残り枚数または経過枚数を表示（例: `45 / 100`）
- 自動モード中も「決まり字を見る」は利用可能

**タブ非表示時:**

- `document.visibilitychange` でタイマーを一時停止し、復帰時に再開する（誤った計測を防ぐ）

v1 から引き継ぐ・変更する点。

| 機能 | v2 での扱い |
|------|-------------|

100 枚完了後に表示。

| 表示項目 | 説明 |
|----------|------|
| 完了メッセージ | 例: 「お疲れさまでした」 |
| 経過時間 | `X分Y秒`（v1 の alert と同等の情報） |

| 「トップへ」 | トップ画面へ |

## 6. PWA 改善

### 6.1 初回インストール案内

#### 6.1.1 表示条件

| 条件 | 説明 |
|------|------|
| 初回のみ | `localStorage` の `pwa_install_prompt_dismissed` が未設定 |
| 未インストール | `display-mode: standalone` ではない（ブラウザ表示中） |
| 対応ブラウザ | `beforeinstallprompt` が発火した場合、または iOS Safari 向け手順表示 |

#### 6.1.2 表示内容

- モーダルまたはバナー
- プラットフォーム別の手順:
  - **Android / Chrome:** 「ホーム画面に追加」ボタン（`beforeinstallprompt`）
  - **iOS Safari:** 共有 → 「ホーム画面に追加」のスクリーンショット付き手順
  - **デスクトップ:** アドレスバーのインストールアイコン案内
- 「後で」「閉じる」で非表示 → `pwa_install_prompt_dismissed = true` を保存

#### 6.1.3 表示タイミング

トップ画面初回表示時（練習開始前）。UX を損なわないよう、0.5〜1 秒遅延後に表示してもよい。

### 6.2 PWA 自動アップデート

#### 6.2.1 現状の課題

- `sw.js` の `CACHE_NAME` を手動更新する必要がある
- `skipWaiting` / `clients.claim` が未実装
- キャッシュファーストのため、古いアセットが残り続ける
- ユーザーが手動でキャッシュ削除・再インストールが必要

#### 6.2.2 目標

デプロイ後、ユーザーの PWA が**自動的に最新版を取得・適用**する。

#### 6.2.3 実装方針

`vite-plugin-pwa`（Workbox）を採用する。

```
1. ビルドごとに Service Worker のハッシュが変わる
2. 新 SW がバックグラウンドで install（waiting 状態）
3. アプリ起動時に updatefound を検知
4. 画面上部などに更新バナーを表示
5. ユーザーがバナーをタップ → skipWaiting → location.reload()
6. 古いキャッシュは Workbox が自動削除
```

**更新 UX（決定）: 起動時通知 + タップで更新**

| 項目 | 仕様 |
|------|------|
| 検知タイミング | アプリ起動時（Service Worker 登録後に `registration.update()` を実行） |
| 表示 | 画面上部のバナーまたはトースト。「新しいバージョンがあります」+ タップで更新 |
| タップ時 | `skipWaiting` を SW に送信 → `controllerchange` イベント後に `location.reload()` |
| 練習中 | バナーは表示してもよいが、**タップによるリロードは練習未開始時のみ**有効。練習中は「練習終了後に更新できます」等の無効表示、またはバナー非表示 |
| 却下 | バナーに「×」で閉じることは可能。次回起動時に再表示 |

バックグラウンドでの新 SW ダウンロードは自動。ユーザー操作なしでの強制リロードは行わない。

#### 6.2.4 Service Worker 戦略

| リソース | 戦略 |
|----------|------|
| HTML / JS / CSS | NetworkFirst または StaleWhileRevalidate |
| 札画像（`torifuda/`） | CacheFirst（容量大・変更頻度低） |
| `manifest.json` | NetworkFirst |
